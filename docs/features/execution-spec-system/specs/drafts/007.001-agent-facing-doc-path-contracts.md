# Execution Spec: 007.001-agent-facing-doc-path-contracts

## Purpose

Update every agent-facing Foundry instruction file so agents use the modular docs layout introduced by `007-modular-docs-feature-layout.md`.

This spec prevents LLMs and Codex from continuing to read, write, or validate the old docs paths after the framework path contract changes.

## Depends On

- `007-modular-docs-feature-layout.md`

Do not execute this spec until 007 is complete.

## Scope

Update all agent-facing instruction surfaces, including at minimum:

- `AGENTS.md`
- `APP-AGENTS.md`
- any copied/scaffolded agent instruction stubs
- any skill files or skill templates committed in the repository
- any docs that instruct agents where feature docs, decision ledgers, execution specs, drafts, or implementation plans live

## Required Path Contract

Replace references to the old structure:

```text
docs/features/<id>-<slug>.md
docs/features/<id>-<slug>.decisions.md
docs/features/<id>-<slug>.spec.md
docs/features/<feature>/specs/<id>-<slug>.md
docs/features/<feature>/specs/drafts/<id>-<slug>.md
```

with the new structure:

```text
docs/features/<feature>/<id>-<slug>.md
docs/features/<feature>/<id>-<slug>.decisions.md
docs/features/<feature>/<id>-<slug>.spec.md
docs/features/<feature>/specs/<id>-<slug>.md
docs/features/<feature>/specs/drafts/<id>-<slug>.md
docs/features/<feature>/plans/<id>-<slug>.md
```

## Required Rules

1. Feature context, feature spec, and decision ledger files are feature-local.
2. Execution specs are feature-local under `docs/features/<feature>/specs/`.
3. Draft execution specs are feature-local under `docs/features/<feature>/specs/drafts/`.
4. Implementation plans are feature-local under `docs/features/<feature>/plans/`.
5. Agents must not create new files under `docs/features/`.
6. Agents must not create new files under `docs/features/<feature>/specs/`.
7. Any remaining references to the old structure must be historical only and clearly marked as such.

## Implementation Steps

1. Search the repository for all old path references:
   - `docs/features`
   - `docs/features/`
   - `features/<feature>`
   - `specs/<feature>`
   - `.decisions.md`
   - `.spec.md`
2. Update agent-facing rules to use the new modular path contract.
3. Update APP-level instructions separately from framework-level instructions.
4. Update any generated/scaffolded agent files so new apps receive the new rules.
5. Preserve existing discipline rules unless they conflict with the new path contract.
6. Do not relax determinism, frozen contract, coverage, or implementation-log requirements.

## Acceptance Criteria

- `AGENTS.md` describes only the new docs layout.
- `APP-AGENTS.md` describes only the new docs layout where relevant.
- Scaffolded agent instruction files use only the new docs layout.
- Repository skills and skill templates use only the new docs layout.
- No active instruction tells agents to write to `docs/features/`.
- No active instruction tells agents to write execution specs to `docs/features/<feature>/specs/`.
- Historical references, if retained, are explicitly labeled as migration history.

## Verification

Run the full relevant test suite and all available docs/spec validation commands.

Also run text searches proving no active old-path instructions remain:

```bash
grep -R "docs/features" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
grep -R "docs/features/<feature>" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
grep -R "docs/features/" AGENTS.md APP-AGENTS.md docs stubs skills README.md APP-README.md 2>/dev/null || true
```

Any remaining matches must be either removed or explicitly marked as historical migration notes.
