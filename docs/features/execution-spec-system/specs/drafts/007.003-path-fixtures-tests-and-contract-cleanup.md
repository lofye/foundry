# Execution Spec: 007.003-path-fixtures-tests-and-contract-cleanup

## Purpose

Update fixtures, tests, snapshots, contract expectations, and deterministic output assertions that depend on the old docs layout.

This spec closes the gap between implementation path changes and test/contract surfaces.

## Depends On

- `007-modular-docs-feature-layout.md`
- `007.001-agent-facing-doc-path-contracts.md`
- `007.002-readmes-stubs-and-public-docs-path-contracts.md`

Do not execute this spec until all dependencies are complete.

## Scope

Update all test and contract surfaces, including at minimum:

- tests that create or assert docs paths
- fixtures containing `docs/features` or `docs/features/<feature>`
- snapshots containing old paths
- JSON contract fixtures
- CLI output expectations
- validation error examples
- smoke/demo app fixtures
- docs import/export tests, if present

## Required Path Contract

Valid current paths:

```text
docs/features/<feature>/<id>-<slug>.md
docs/features/<feature>/<id>-<slug>.decisions.md
docs/features/<feature>/<id>-<slug>.spec.md
docs/features/<feature>/specs/<id>-<slug>.md
docs/features/<feature>/specs/drafts/<id>-<slug>.md
docs/features/<feature>/plans/<id>-<slug>.md
```

Invalid active paths:

```text
docs/features/*
docs/features/<feature>/specs/*
```

## Implementation Steps

1. Search all tests, fixtures, snapshots, and contract files for old path references.
2. Update fixture directory structures to mirror the new modular layout.
3. Update CLI expected outputs to emit new paths.
4. Update validation failure messages to reference new paths.
5. Update deterministic JSON contracts so path fields match the new layout.
6. Remove obsolete fixture directories when they are no longer used.
7. Keep ordering deterministic in all updated outputs.

## Acceptance Criteria

- Tests no longer rely on `docs/features/` for active feature docs.
- Tests no longer rely on `docs/features/<feature>/specs/` for active execution specs.
- Snapshot and JSON fixtures use modular paths.
- CLI contract outputs remain deterministic.
- Validation errors describe the new path contract.
- No obsolete fixture directory is left active by accident.

## Verification

Run:

```bash
php bin/foundry spec:validate --json
php bin/foundry verify contracts --json
php bin/foundry verify graph --json
php bin/foundry verify pipeline --json
```

Run the full test suite.

Also run:

```bash
grep -R "docs/features" tests fixtures snapshots 2>/dev/null || true
grep -R "docs/features/" tests fixtures snapshots 2>/dev/null || true
```

Any remaining matches must be removed or explicitly documented as migration-only fixtures.
