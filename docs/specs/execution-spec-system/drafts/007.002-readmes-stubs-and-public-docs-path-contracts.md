# Execution Spec: 007.002-readmes-stubs-and-public-docs-path-contracts

## Purpose

Update Foundry's human-facing documentation, README files, examples, and stubs so they reflect the modular docs layout introduced by `007-modular-docs-feature-layout.md`.

This spec handles documentation surfaces that are not primarily agent instruction files.

## Depends On

- `007-modular-docs-feature-layout.md`
- `007.001-agent-facing-doc-path-contracts.md`

Do not execute this spec until both dependencies are complete.

## Scope

Update all human-facing and generated documentation surfaces, including at minimum:

- `README.md`
- `APP-README.md`
- `docs/**/*.md`
- `stubs/**/*.stub`
- example apps or smoke/demo docs
- CLI help examples stored in docs or stubs
- any documentation describing feature docs, decision ledgers, execution specs, draft specs, or implementation plans

## Required Path Contract

Current modular layout:

```text
docs/<feature>/<id>-<slug>.md
docs/<feature>/<id>-<slug>.decisions.md
docs/<feature>/<id>-<slug>.spec.md
docs/<feature>/specs/<id>-<slug>.md
docs/<feature>/specs/drafts/<id>-<slug>.md
docs/<feature>/plans/<id>-<slug>.md
```

Old layout, now invalid for active documentation:

```text
docs/features/<id>-<slug>.md
docs/features/<id>-<slug>.decisions.md
docs/features/<id>-<slug>.spec.md
docs/specs/<feature>/<id>-<slug>.md
docs/specs/<feature>/drafts/<id>-<slug>.md
```

## Implementation Steps

1. Search all documentation, stubs, fixtures, and examples for old path references.
2. Update README guidance to describe feature-local documentation.
3. Update APP README guidance to describe app-level expectations without referencing obsolete framework paths.
4. Update stubs so newly generated apps do not contain obsolete paths.
5. Update examples that show creating, validating, promoting, or executing specs.
6. Update any diagrams, tables, or code blocks that show old docs paths.
7. Preserve historical notes only when they are explicitly labeled as historical.

## Acceptance Criteria

- `README.md` contains no active references to `docs/features/`.
- `APP-README.md` contains no active references to `docs/features/`.
- Docs and examples do not instruct users to place execution specs under `docs/specs/<feature>/`.
- Stubs generate only new modular docs paths.
- Any migration notes clearly distinguish old paths from current paths.
- The documentation explains that docs are now feature-local and modular.

## Verification

Run the repository test suite and documentation/spec validation commands.

Also run:

```bash
grep -R "docs/features" README.md APP-README.md docs stubs examples 2>/dev/null || true
grep -R "docs/specs/" README.md APP-README.md docs stubs examples 2>/dev/null || true
```

Any remaining matches must be removed or explicitly marked as historical migration notes.
